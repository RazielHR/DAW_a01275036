Preguntas


¿Cuál es la diferencia entre Internet y la World Wide Web?

De acuerdo a un articulo publicado en BBC News Mundo, "Internet es una una inmensa red de computadoras al rededor de todo el mundo conectdas entre sí"[1]. A esta definición faltaría agregar a los dispositivos moviles, tabletas, y todos los dispositivos que tengan acceso a esta red.

Por otro lado, en el mismo articulo describe a la World Wide Web como: "... la web (la World Wide Web) es una enorme colección de páginas que se asienta sobreesa red de computadoras"[1]. En otras palabras, internet es el ciber espacio, y la World Wide Web son la colección de páginas dentro de internet.

De acuerdo a la página web MDN web docs mozilla, el proposito de los métodos HTTP es "definir un conjunto de metodos de petición para indicar la acción que se desea realizar para un recurso determinado"[2].

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Tomando en cuenta la pregunta anterior y el funcionamiento de cada método, POST sería el método para enviar un formulario en HTML porque se utiliza para enviar una entidad a un recurso en especifico.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

De acuerdo a las descripciones obtenidas sobre los metodos HTTP, GET sería el método que se utiliza, ya que este metodo solicita solicita una representación de un recurso especifico, en este caso la página.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Una respuesta con código 200 indica que hubo éxito en la solicitud HTTP, por lo tanto no es un error[3].

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Una respuesta con código 404 indica que el servidor no encontro el contenido que se solicita, por lo tanto el desarrollador debe crear una página que atrape estos errores y le explique al usuario que no existe el contenido al que intenta acceder[3].
